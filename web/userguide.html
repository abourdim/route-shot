Install

git clone https://github.com/abourdim/route-shot.git
cd route-shot
./menu.sh            # option 2 installs playwright + chromium

Requires Node.js 18+. The menu.sh bash script orchestrates everything; direct node route-shot.js … invocations work too.

Quick start

Single URL, no config file:

node route-shot.js https://example.com

Batch (many apps from a JSON config):

node route-shot.js --batch apps.json

Minimal apps.json:

{
  "apps": [
    {
      "name": "my-app",
      "url": "https://example.com",
      "autoButtons": true
    }
  ]
}
Four fields (name, url, dismiss, autoButtons) cover 95% of use cases.

Interactive menu

bash menu.sh
#Action
1Check install (Node, Playwright, Chromium)
2Install dependencies
3Launch route-shot (single URL prompt)
4Batch run — reads apps.json
5Import DevTools Recorder JSON → apps.json
6Start local web server on localhost:8000 to view screenshots
7Open viewer in browser
8Stop web server
9Open screenshots folder in OS file manager
10Clean screenshots folder
11Exit

apps.json structure

{
  "apps": [
    { /* app 1 */ },
    { /* app 2 */ }
  ]
}

Each app object must have name and url. Everything else is optional.

Strict JSON — no trailing commas, no comments. The last key in an object must not have a , after it.

Two complementary click modes

ModeUse when
clicks: [...] explicitYou know the buttons you want. Full control.
autoButtons: trueYou want every discoverable button. Zero config beyond a blocklist.

You can combine both — explicit clicks run first, then auto-discovery fills in the rest (deduped).

All fields reference

Identity & crawling

FieldTypeDefaultWhat it does
namestring—App name; becomes the output subfolder name
urlstring—Start URL
maxPagesnumber200Cap on same-origin pages the BFS will visit
navTimeoutnumber15000Per-page navigation timeout (ms)

Modal dismissal

FieldDefaultWhat it does
dismissnoneSelector(s) clicked after each page load. Comma-separated or array.

Explicit clicks

FieldDefaultWhat it does
clicks[]Array of selectors. Each is clicked and captured as a variant screenshot.
clickModesequentialsequential — clicks accumulate on same page. independent — reload between each so state doesn't leak.
clickWait500ms to wait after each click before screenshotting

Auto-discovery

FieldDefaultWhat it does
autoButtonsfalseDiscover every button / [role=button] / input[type=button] / input[type=submit] and click each.
autoButtonsSelectorsee aboveOverride the discovery selector
autoButtonsBlocklistdelete|remove|clear|stop|reset|pay|confirm|send|logout|sign.?outRegex (case-insensitive) — any button whose visible text matches is skipped
autoButtonsMax40Safety cap on buttons per page

Advanced

FieldDefaultWhat it does
preSteps[]Setup steps (login, expert-mode toggle). Replayed after every independent reload.
maxDepth0Recursive exploration depth. 1 = click A then discover and click more buttons on the new state.
headfulfalseShow Chromium window — useful for debugging

Examples

Simplest possible

{
  "apps": [
    { "name": "my-site", "url": "https://my-site.com" }
  ]
}

→ Crawls same-origin <a href> links, one screenshot per page. No clicking.

Single page, auto-click everything

{
  "apps": [
    {
      "name": "bit-playground",
      "url": "https://abourdim.github.io/bit-playground/",
      "dismiss": "button:has-text(\"Let's Go\")",
      "autoButtons": true,
      "maxPages": 1
    }
  ]
}

Explicit list of buttons

{
  "apps": [
    {
      "name": "bit-playground-tabs",
      "url": "https://abourdim.github.io/bit-playground/",
      "dismiss": "button:has-text(\"Let's Go\")",
      "clickMode": "independent",
      "clicks": [
        "#controlsTabBtn",
        "#sensesTabBtn",
        "#gamepadTabBtn",
        "#graphTabBtn"
      ]
    }
  ]
}
independent reloads between each click, so every screenshot shows one tab active from a fresh state.

Click a specific button at start, then auto-sweep

{
  "apps": [
    {
      "name": "bit-playground-expert",
      "url": "https://abourdim.github.io/bit-playground/",
      "dismiss": "button:has-text(\"Let's Go\")",
      "preSteps": [
        { "action": "click", "selector": "button:has-text(\"expert\")" }
      ],
      "autoButtons": true,
      "autoButtonsBlocklist": "🌙|⚡|☁️|🔥|beginner",
      "maxPages": 1
    }
  ]
}

Many apps in one batch

{
  "apps": [
    { "name": "home",    "url": "https://my-site.com/",    "autoButtons": true },
    { "name": "docs",    "url": "https://my-site.com/docs", "autoButtons": true },
    { "name": "pricing", "url": "https://my-site.com/pricing" }
  ]
}

preSteps action reference

Each entry in preSteps is an object with an action field. Runs after goto + dismiss, replays on every independent reload.

ActionFieldsExample
gotovalue / url{ "action": "goto", "value": "https://x.com/login" }
clickselector{ "action": "click", "selector": "#signInBtn" }
fillselector, value{ "action": "fill", "selector": "#email", "value": "a@b.com" }
pressselector, value (key){ "action": "press", "selector": "#q", "value": "Enter" }
waitForSelectorselector, timeout?{ "action": "waitForSelector", "selector": ".dashboard" }
waitForURLvalue / url, timeout?{ "action": "waitForURL", "value": "**/dashboard" }
waitvalue / ms{ "action": "wait", "value": 1000 }

Login flow example

{
  "name": "app-behind-login",
  "url": "https://app.example.com",
  "preSteps": [
    { "action": "fill",       "selector": "input[name=email]",    "value": "demo@example.com" },
    { "action": "fill",       "selector": "input[name=password]", "value": "hunter2" },
    { "action": "click",      "selector": "button:has-text(\"Sign in\")" },
    { "action": "waitForURL", "value": "**/dashboard" }
  ],
  "autoButtons": true
}

Session cookies and localStorage persist across the browser context, so subsequent page navigations stay authenticated.

Chrome DevTools Recorder import

  1. DevTools → Recorder panel → Start new recording
  2. Click through the flow you want captured
  3. End recording → Export → "Export as a JSON file" (NOT Puppeteer or Playwright — those are JS scripts route-shot can't parse)
  4. Save as flow.json

Then convert:

# print the translated entry to stdout
node route-shot.js --import-recording flow.json

# or merge directly into apps.json (upsert by name)
node route-shot.js --import-recording flow.json --name my-app --merge apps.json

# treat the recorded clicks as variants (one screenshot per click) instead of one-shot preSteps
node route-shot.js --import-recording flow.json --as-clicks --merge apps.json
FlagMeaning
--import-recording <path>Source Recorder JSON
--name <name>Override app name (default: recording title)
--url <url>Override start URL
--as-clicksMap clicks to clicks[] instead of preSteps[]
--merge <apps.json>Upsert into the config file
The importer picks the most stable selector candidate from Recorder's list — prefers #id over aria/…, xpath/…, pierce/…, text/… alternatives.

Writing blocklist patterns

autoButtonsBlocklist is a regex tested against each button's visible text. Case-insensitive. | means OR.

GoalPattern
Block exactly "Reset""^Reset$"
Block any button containing "reset""reset"
Block multiple texts"delete|pay|confirm"
Block emoji-only buttons"🌙|⚡|☁️|🔥"
Block nothing (full sweep)"^$"
Block literal question mark"\\?"

Finding the right text

Run batch once, open screenshots/<app>/index.json:

"variants": [
  { "label": "🌙",        "auto": true, "screenshot": "..._emoji_12065.png" },
  { "label": "Firmware",  "auto": true, "screenshot": "..._Firmware.png"   },
  { "label": "Controls",  "auto": true, "screenshot": "..._Controls.png"   }
]

Copy any label values you don't want and glue them with |.

Tips & gotchas

Don't crawl, just snapshot — if you only want one screenshot of the landing URL, set "maxPages": 1. Otherwise BFS follows every <a href>.
Modal blocks your preSteps? route-shot now dismisses before preSteps, but if your modal uses a different selector than the one you think, the click will silently fail. Enable "headful": true and watch it happen live.
autoButtons captures way too many near-duplicates? Many buttons on the page may only toggle a tiny visual detail. Use clicks[] with an explicit list instead.
Independent mode is expensive — reloads the page for every single click. With 40 buttons that's 40 full page loads. Budget accordingly or drop autoButtonsMax.
Playwright has-text is substring — has-text("Clear") matches both "Clear" and "Clear All". If you need exact match, use text-is("Clear") or role-based getByRole('button', { name: 'Clear', exact: true }) (autoButtons already does this).
Never run autoButtons on a production app with real data. Even with a blocklist, you can't be sure "Save" won't trigger something you don't want. Use a staging env or the default blocklist as a minimum.
preSteps don't replay after goto in BFS — they replay after every independent-mode reload (auto-button variants), but not when the BFS crawls to a new URL. Same-app multi-page flows may lose preSteps state.
Session persists — a single browser context is used for the whole app, so cookies and localStorage survive page navigations. Log in once in preSteps, crawl all pages authenticated.

Troubleshooting

"Element is not visible, enabled and stable"

A modal or overlay is covering the target. Check dismiss selector. Bump dismissWait. Try "headful": true to see it.

"Not found" / "elementHandle.click: Timeout"

Selector doesn't match anything. Two reasons:

  • CSS is case-sensitive for IDs — #ExpertBtn ≠ #expertBtn
  • Auto-discovery uses exact role-name match — if the label has weird whitespace or multiple elements share it, disambiguate with an explicit selector in clicks[]

"Unexpected token … is not valid JSON"

Usually a trailing comma. Last key in an object, last item in an array — no comma after it.

{
  "foo": 1,
  "bar": 2,   ← OK (not last)
  "baz": 3    ← no comma!
}

Importer says "looks like a Puppeteer/Playwright script export"

You exported as Puppeteer/Playwright JS instead of JSON. In the Recorder panel, pick "Export as a JSON file" (top of the list).

Top-nav tabs aren't clicked

Happens when multiple elements share the tab text (e.g. a heading plus the button). Fix: add the tab buttons to clicks[] using ID selectors like #controlsTabBtn. Auto-discovery will skip them thanks to dedup.

Welcome modal keeps coming back every screenshot

Independent mode reloads for every click, so the modal re-renders each time. Dismiss handles this. If it's still visible in screenshots, either dismissWait is too short or the selector is wrong.

Glossary

BFSBreadth-first search. route-shot's link crawling: visits the start URL, then every <a href> it finds, then theirs, and so on — level by level.
VariantA screenshot of the same URL after a click. Captured as ..._v01_<label>.png.
Independent modeEach click starts from a fresh reload. Prevents state leaking between clicks. Slower but cleaner.
Sequential modeClicks accumulate. Matches tab-like navigation where state should persist.
preStepsSetup actions (login, toggles) run once at start and replayed on every independent reload.
dismissSelector(s) clicked immediately after every page load — typically onboarding modals, cookie banners.
RecorderChrome DevTools' Recorder panel. Records user interactions, exports to JSON / Puppeteer / Playwright. route-shot only consumes the JSON.
BlocklistRegex of button texts to skip during autoButtons discovery. Prevents clicking destructive actions.
