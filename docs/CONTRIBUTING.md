# Contributing

Thanks for wanting to help. This is a small project — keep it simple.

## Reporting bugs

Open an issue with:
- what you ran (OS, Node version, start URL if you can share it)
- what you expected
- what actually happened (paste the terminal output)

## Proposing changes

1. Fork the repo
2. Create a branch: `git checkout -b feat/short-name`
3. Keep the change focused — one feature or fix per PR
4. Test it against at least one real site (or a local HTML file)
5. Open a PR describing what changed and why

## Style

- Node 18+ features are fine
- No build step, no TypeScript, no framework — the crawler is one file on purpose
- Comments only where the code can't speak for itself

## Ideas welcome

See the roadmap in [README.md](README.md). Open an issue to discuss before building anything big.
